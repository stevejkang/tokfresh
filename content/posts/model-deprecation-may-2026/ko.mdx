## 무슨 일이 있었나

Anthropic이 `claude-sonnet-4-20250514` 모델의 지원을 종료했습니다. 예전 TokFresh 워커들이 핑 요청에 쓰던 모델입니다. Anthropic의 [모델 지원 종료 페이지](https://platform.claude.com/docs/about-claude/model-deprecations)에 공지되었고, 유예 기간 없이 지원 종료가 발효되자마자 해당 모델로 가는 요청은 아예 작동을 멈췄습니다.

## 영향

이 변경 이전에 워커를 배포하셨다면, 예약된 핑 요청에서 404 오류가 발생했을 수 있습니다. 요청 자체엔 문제가 없었습니다. 요청하려던 모델이 Anthropic 쪽에서 더 이상 존재하지 않게 된 것뿐입니다. 최근 리셋 주기가 예정대로 트리거되지 않았다면, 이게 거의 확실한 원인입니다.

모든 워커가 똑같이 영향받은 건 아닙니다. 배포 시점에 따라 워커 코드에 박힌 모델 버전이 달랐기 때문인데, 배포한 지 꽤 됐고 그 이후 재배포한 적이 없다면 영향을 받았다고 보시면 됩니다.

## 바뀐 것

세 가지를 변경했습니다. 두 가지는 즉각적인 조치였고, 하나는 구조적인 개선이었습니다.

**모델 교체.** TokFresh는 이제 지원이 종료된 Sonnet 대신 핑 요청에 `claude-haiku-4-5-20251001`을 사용합니다. Haiku는 더 가볍고 빠르며, 핑 요청이 의미 있는 출력이 아니라 사용 주기 트리거만 담당한다는 점에서 Sonnet보다 오히려 잘 맞는 선택입니다.

**동적 설정.** 모델 이름과 API 헤더는 더 이상 배포 시점에 하드코딩되지 않습니다. 이제 실행될 때마다 TokFresh 서버에서 가져오며, 짧은 캐시와 안전한 폴백이 함께 적용됩니다. [별도 포스트](/posts/dynamic-model-and-header-fetching)에서 자세히 다뤘습니다만, 간단히 말해 이 문제가 다시 발생하는 걸 막아주는 근본적인 해결책입니다.

**앞으로의 자동 대응.** 동적 설정 덕분에 다음에 Anthropic이 모델을 지원 종료하거나 API 사양을 바꾸더라도, 엔드포인트 하나만 업데이트하면 재배포된 모든 워커가 다음 예약 실행 때 자동으로 맞춰집니다. 새 공지도, 재배포도, 조용한 404 반복도 필요 없습니다.

## 지금 하셔야 할 일

이 변경 이전에 워커를 배포하셨다면, 한 번만 재배포하시면 됩니다.

1. [tokfresh.com/setup](https://tokfresh.com/setup)으로 이동합니다
2. 원래 사용하셨던 계정으로 로그인합니다
3. 기존 스케줄과 설정은 그대로 둡니다
4. Deploy를 클릭합니다

새 워커가 Cloudflare 계정의 기존 워커를 자동으로 대체하며, 별도의 정리 작업은 필요 없습니다.

영향을 받았는지 확실하지 않으셔도, 재배포는 어느 쪽이든 해가 되지 않습니다.

## 그 이후

재배포를 마치셨다면 앞으로 문제없습니다. 이번이 모델 지원 종료로 인해 직접 조치해야 하는 마지막 사례가 되는 게 목표입니다. 동적 설정 시스템이 현재 모델과 헤더 값을 자동으로 가져오기 때문에, 앞으로의 변경 사항은 또 다른 공지나 재배포, 리셋 스케줄의 공백 없이 흡수됩니다.

## 투명성

모델 교체, 동적 설정 fetch, 폴백 로직 모두 [github.com/stevejkang/tokfresh](https://github.com/stevejkang/tokfresh)의 TokFresh 저장소에 공개되어 있습니다. 재배포 전에 코드를 직접 확인하실 수 있습니다.
